# 🌐 T02: Creant la proposta de pàgina corporativa

## 📝 Breu descripció

### 📌 Introducció al cas
Una de les primeres inquietuds que us han fet arribar des de **FoodLogistic S.A.** és que la seva **presència a Internet ha de millorar**.

Per una banda, la **pàgina web actual** s’ha quedat:
- 🎨 Estèticament desactualitzada  
- 🛠️ Tècnicament obsoleta  

A més, **no compleix la normativa vigent**:
- 📄 **LOPDGDD**
- 🌍 **LSSI-CE**

Per aquest motiu, us demanen una **proposta de nova web corporativa**.  
❗ No volen una prova funcional local, per tant caldrà **publicar el contingut en un servidor real** per demostrar al client com serà la seva **nova identitat digital**.

En aquesta activitat posareu en pràctica la **guia de desplegament** disponible per assolir **tres objectius clau**:

---

## 🎯 Objectius de l’activitat

### 🚀 Publicació professional
- Utilitzareu **GitHub Pages**
- Convertireu el vostre codi en una **web operativa**
- Disponible sota una **URL pública**

### 📚 Documentació del projecte
- Crear un fitxer **README.md professional**
- Ha de servir com a:
  - Carta de presentació de la vostra **empresa**
  - Presentació del seu **autor**

### 📊 Control i analítica
> No es pot millorar el que no es mesura

- Integrareu una eina de control web
- 🕵️ **StatCounter**
- Permetrà saber **qui visita la web** de la vostra empresa

---

## ✅ Punts clau

- Crear un **repositori** al vostre **GitHub personal**
- Documentar-lo correctament amb un **README.md**
- GitHub Pages requereix una **estructura específica**:
  - 📁 Carpeta `/docs`
  - 📄 Fitxer `index.html`
  - 🌐 Resta d’objectes de la web dins `/docs`

- Configurar la **branca principal (`main`)**:
  - Serà la que **actualitza la web**
  - Cada cop que es confirma un canvi

⚠️ Recomanació important:
- ✍️ Treballar **en local**
- Fer els **commits necessaris**
- Pujar a GitHub **només** quan hi hagi:
  - Una **nova versió estable** a desplegar

- Implementar un **“Comptador Invisible”**:
  - 📈 **StatCounter**
  - Per fer seguiment de les visites
